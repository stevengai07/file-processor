# 批量上传与处理

> 路由建议：`/tasks/new`、`/tasks/{task_id}`  
> 模块：批量任务  
> 日期：2026-06-09

## 页面目标

用户使用已确认的字段模板批量上传文档，配置提取参数，启动任务并查看每份文件的实时处理状态。

## 页面布局

1. 顶部：模板名称、字段数量和“返回修改模板”。
2. 文档上传区：多文件拖放、文件列表和校验结果。
3. 提取设置区：模型、OCR 和语言。
4. 任务状态区：总进度和状态统计。
5. 文件队列表：逐文件状态、耗时、错误和操作。
6. 底部操作区：开始提取、取消未开始任务、查看结果。

## 提取设置

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|---|---|---:|---|---|
| 模型供应商 | 下拉选择 | 是 | 当前已配置供应商 | OpenAI、Anthropic、Google、DeepSeek、xAI、Alibaba |
| 模型 | 下拉选择 | 是 | 性价比默认模型 | 仅显示供应商支持的模型 |
| OCR 预设 | 下拉选择 | 是 | scanner | scanner、phone_camera、faded、fax |
| OCR 语言 | 下拉选择 | 是 | eng | 使用现有 Tesseract 语言集合 |
| 混合英文 OCR | 开关 | 否 | 关 | 非英文文档混合英文时启用 |
| 并发数 | 数字步进器 | 否 | 4 | 上限根据部署资源配置 |

## 文件队列表

| 列 | 说明 |
|---|---|
| 文件名 | 原始文件名 |
| 类型 | PDF 或 DOCX |
| 大小 | 格式化后的文件大小 |
| 状态 | 等待、处理中、成功、待检查、失败、已取消 |
| 进度 | 当前文件处理阶段 |
| 耗时 | 从开始到结束的时间 |
| 问题 | 缺失字段数或错误摘要 |
| 操作 | 删除、重试、查看结果 |

## 交互

### 添加文件

- 支持一次选择多个文件和多次追加。
- 同名同大小文件默认判定为重复，提示用户跳过或保留。
- 不支持的格式、超限文件在上传阶段标红，不进入处理队列。
- 开始任务后不允许向当前任务追加文件；需要新增任务或复制任务 `[TBC]`。

### 开始提取

- **前置条件：** 模板有效、至少一个合法文件、模型 API 密钥可用。
- **响应：** 创建任务，合法文件进入等待队列，系统按并发数处理。
- **处理阶段：** 文件读取 → 原生文本提取/OCR → AI 字段抽取 → 类型校验 → 状态判定。

### 状态判定

- 所有字段结构合法且必填字段齐全：成功。
- 提取完成但存在必填缺失或类型错误：待检查。
- 文件读取、OCR、模型请求或结构解析失败：失败。
- 用户取消尚未开始的文件：已取消。

### 失败重试

- 用户可以对单个失败文件点击“重试”。
- 重试沿用当前任务模板和设置。
- 每次重试都更新错误信息；是否保留完整尝试历史为 `[TBC]`。

### 修改模板

- 尚未启动任务时可返回模板页面修改。
- 已启动后修改模板必须创建新模板版本，并明确询问是否重新处理全部文件。

## API 依赖

| API | 方法 | 路径 | 用途 |
|---|---|---|---|
| 创建任务 | POST | `/api/tasks` | 绑定模板和处理设置 |
| 上传文档 | POST | `/api/tasks/{task_id}/files` | 批量上传文档 |
| 启动任务 | POST | `/api/tasks/{task_id}/start` | 开始后台处理 |
| 查询任务 | GET | `/api/tasks/{task_id}` | 获取总状态和进度 |
| 查询文件 | GET | `/api/tasks/{task_id}/files` | 获取文件级状态 |
| 重试文件 | POST | `/api/tasks/{task_id}/files/{file_id}/retry` | 重试失败文件 |
| 取消任务 | POST | `/api/tasks/{task_id}/cancel` | 取消等待中的处理 |

## 验收标准

- 前端可一次加入多个 PDF/DOCX。
- 任务运行时可看到总进度和逐文件状态。
- 单文件失败不会中断其他文件。
- 失败文件可重试。
- 完成后可以进入结果页，并保留模板字段顺序。
