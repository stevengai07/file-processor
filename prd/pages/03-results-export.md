# 结果校对与导出

> 路由建议：`/tasks/{task_id}/results`  
> 模块：结果管理  
> 日期：2026-06-09

## 页面目标

以动态表格展示所有文档的字段提取结果，帮助用户定位缺失或错误内容、人工修正，并将最终结果导出为 Excel 或 DOCX。

## 页面布局

1. 顶部统计：文档总数、成功、待检查、失败。
2. 工具栏：状态筛选、文件名搜索、仅看有问题字段、导出。
3. 结果数据表：固定元数据列 + 模板动态字段列。
4. 右侧详情面板：单份文档的全部字段、校验问题和原文出处。
5. 导出对话框：格式、范围和附加选项。

## 结果表字段

| 列 | 类型 | 说明 |
|---|---|---|
| 选择 | 复选框 | 用于批量导出 |
| 文件名 | 文本 | 原始文件名 |
| 状态 | 状态标签 | 成功、待检查、失败 |
| 问题数 | 数字 | 必填缺失、类型错误等数量 |
| 模板字段列 | 动态 | 按模板顺序生成，每个字段一列 |
| 最后修改 | 日期时间 | AI 完成或人工修改时间 |
| 操作 | 图标按钮 | 查看详情、重试 |

## 交互

### 查看和筛选

- 页面默认展示成功和待检查记录，失败记录保留在同一任务中。
- 支持按文件名模糊搜索。
- 支持按状态筛选。
- “仅看有问题字段”开启后，详情面板聚焦缺失和类型错误字段。

### 编辑字段

- 用户点击单元格或详情字段进入编辑状态。
- 保存时按字段数据类型校验。
- 编辑成功后标记该字段“已人工修改”。
- 当所有必填问题解决后，记录状态从“待检查”变为“成功”。
- 需要保存修改人时必须先引入账号系统；MVP 无账号时仅记录修改时间。

### 查看来源

- MVP 建议保存字段对应的原文片段，详情面板可展示。
- 当前提取链路仅返回字段值，不返回来源和置信度，因此该能力需要扩展模型输出。
- 若 MVP 不实施来源追踪，页面不展示虚假置信度，标记为后续功能。

### 导出 Excel

- 默认范围：成功和待检查记录。
- 可选择全部记录或勾选记录。
- 主工作表“提取结果”：一份文档一行，模板字段一列。
- 辅助工作表“处理记录”：文件名、状态、错误、处理时间、模型。
- 保持模板字段顺序，人工修改值优先。

### 导出 DOCX

- 生成一个汇总文档。
- 首页包含任务名称、导出时间和统计。
- 每份文档一个一级标题，每个字段按模板顺序展示。
- 失败文件放在末尾“处理失败记录”章节，不生成空业务字段章节。
- “使用 Word 模板占位符回填”不在默认 MVP 中，是否需要为 `[TBC]`。

## API 依赖

| API | 方法 | 路径 | 用途 |
|---|---|---|---|
| 查询结果 | GET | `/api/tasks/{task_id}/results` | 分页、搜索和筛选结果 |
| 获取单份详情 | GET | `/api/tasks/{task_id}/results/{file_id}` | 获取字段、问题和来源 |
| 修改字段 | PATCH | `/api/tasks/{task_id}/results/{file_id}` | 保存人工校对结果 |
| 导出 Excel | POST | `/api/tasks/{task_id}/exports/excel` | 生成并下载 XLSX |
| 导出 DOCX | POST | `/api/tasks/{task_id}/exports/docx` | 生成并下载 DOCX |

## 验收标准

- 动态结果列与模板字段完全一致且顺序一致。
- 用户可编辑字段并通过类型校验。
- 修改后的值用于 Excel 和 DOCX 导出。
- Excel 至少包含“提取结果”和“处理记录”两个工作表。
- DOCX 按文档分节并列出所有字段。
- 导出失败不会影响页面中的结果数据。
